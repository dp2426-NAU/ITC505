Puzzle Game

Addendum

Process:

The objective of the project was to create a puzzle game with a random starting configuration that is solvable. The first step was to design a basic 5x5 grid layout to represent the puzzle board. Each square toggles between two colors—white (the "off" state) and black (the "on" state)—whenever clicked. I used a grid layout in CSS to keep the design clean and simple, as it allows for easy placement and management of the square elements.

After setting up the basic layout, I focused on creating the game’s logic. I began by implementing a function that would toggle the state of a square, as well as its neighboring squares (up, down, left, right) when clicked. This mechanic mirrors the core concept of the "Lights Out" puzzle, where turning one light on or off also affects adjacent lights.

Creative Choices:

  • Grid Size: I chose a 5x5 grid because it strikes a balance between simplicity and challenge. A smaller grid might have been too easy, while a larger one could make the game frustrating for players.
  • Color Scheme: The colors white and black were chosen for simplicity and contrast. White represents the "off" state (goal state) and black represents the "on" state. The toggling between these colors is visually clear and immediate, which aids in gameplay.
  • Alert System: I implemented the "You win!" message using the window.alert() method, which provides an immediate and clear indication to the user once they have solved the puzzle. This ensures the player knows the outcome without ambiguity.
